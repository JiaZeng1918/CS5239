1 Form a team of 5 students – before 8 Oct, midnight.

Under Canvas People, you can find a section titled “Teams”. Join a team under this section. The maximum number of students in a team is 5, and the minimum number of students in a team is 4. Use your team number to identify your team later on.

Students that do not choose a team before the team formation deadline will be randomly allocated to a team.

When you submit your work in Canvas, you will submit as a team. The latest submission by any of the team members will be graded, and the grade will be the same for all team members.

1.1 Setup a method of collaboration for your team

You might use Google Drive or a repository in GitHub. Choose a team leader that will book a presentation slot, and complete the administrative part of the submissions for the whole team.

Beware that you should not use public repositories for your work. make sure that your repository is private.

If you need to create a private Github repository, you can obtain one using Github Classroom by visiting the project link. The first member of the team to access this link will be prompted to accept the assignment that gives your team access to the new repository. Create a new team by typing teamXX where XX stands for your team number from Canvas. For example, team 8 should name the repository team08. Use the same name for your Canvas submission for easy identification. The other member in the team will be able to see an existing team with your team id under “Join an existing team” section. The repository is empty. Use this repository for any scripts or code. It is not mandatory to create a repository for your team within Github Classroom.

2 Project (35%) – due on Tue, 15 Nov, 6:30pm

We proposed seven projects for you. You may find their write-ups in Canvas Files, as html files. The projects topics are shown below:

  • Project 1 - CPU Monitoring (Linux)
  • Project 2 - Memory Monitoring (Linux)
  • Project 3 - I/O Monitoring (Linux)
  • Project 4 - Network Monitoring (Linux)
  • Project 5 - I/O, Memory, CPU, and Network Monitoring (Windows)
  • Project 6 - Data Plots and Visualization (Colab)
  • Project 7 - Performance of Machine Learning Applications (Colab)

Each project has a some guided tasks and ends with at least one open-ended task. You should choose one of the projects to work on it as a team. At the end of the semester, you are required to submit under Canvas Assignments –> Project, a report PDF file (teamXX.pdf) or a zip archive containing:

  • a report (teamXX.pdf) - solving the guided tasks and the open-ended task of the chosen project
  • any additional scripts or code that you have used for your project

For the Linux projects, you should try to install a Linux distribution natively on your computer. A Linux distribution running in a virtualization environment (VM - virtual machine) will not provide measurements for most of the counters used for performance measurement. If you can’t install Linux natively, but you are working on a Linux-based project, you can use a set of machines located in the School of Computing. Details about how to access these machines can be found in Appendix.

2.1 Content requirements

  • You are expected to expand and develop your project based on the open-ended tasks. A considerable part of your project report must focus on performance analysis of the open-ended task.
  • You can use any tools you want, even if they are not specified in the project writeup.
  • Use graphs (plots) and visual diagrams to explain and illustrate your points.
  • Do not include code snippets and scripts in your report, unless they are essential for the understanding of the content. When appropriate, you can include commands that were useful in your project.
  • You are allowed to propose and analyse a different open-ended task with prior approval from Cristina (ccris@comp.nus.edu.sg). The newly proposed task must be related to the big topic of the project.

2.2 Project report guidelines

  • Mention which project you attempted in the title (using the project number).
  • Maximum 20 pages long in PDF format.
  • All text in your report should be no smaller than 11-point Arial (any typeface and size is ok so long as it’s readable English and not trying to bypass the page limit).
  • All page margins (top, bottom, left, right) should be at least 1 inch (2.54cm) and have visually distinct headers for each task.
  • It should be self-contained. If you write part of your report somewhere else and reference that in your submitted report, we reserve the right to ignore any content outside the submitted document. An exception is referencing a document containing measurement data that you created as part of the assignment - we encourage you to do this.
  • If headers, spacing or diagrams cause your report to slightly exceed the page limit, that’s ok - we prefer well-organised, easily readable reports.

5.1 Connecting to the computers in SoC

You are given access to the machines in the Parallel and Distributed Computing Lab located in COM1-B1-02. To connect to these computers, you should first use your SoC id (Unix id) to connect to stu.comp.nus.edu.sg. If you do not have an SoC id, you can create one at this link. With your Unix id, eg soc_unix_id, connect to stu.comp.nus.edu.sg with:

> ssh soc_unix_id@stu.comp.nus.edu.sg

Now, you can remotely connect to any of the machines in the lab. These run Ubuntu 20.04.3 LTS on the x86_64 architecture. The hostnames span the range: soctf-pdc-001 to soctf-pdc-024.

Direct login is not enabled to all lab machines because some nodes are managed by Slurm Workload Manager. You can use the nodes that have direct login enabled. As of 6 Oct, direct ssh access is disabled to soctf-pdc-[005-008,013-016,019,021] You can use Slurm to submit your batch jobs, as well. Updated Slurm guide can be found here.

From within stu, use the username and password provided to you in Canvas Gradebook (under Username and Password field, you will find a comment) to connect to any of the lab machines mentioned above. For example, if your username is username, and you choose soctf-pdc-011, you can do:

> ssh username@soctf-pdc-011

You can change your default password by running passwd in the console. Note that each account on each machine is independent (there is no automatic synchronization among machines). This means that changing the password on one machine will not change the passwords on the other machines. Also, files placed on the home directory of one machine will not be automatically copied to other machines (you need to manually copy them). Use scp, wget or git to transfer files to and from the lab machines.

To login without inputting a password, configure SSH key-based authentication. Use ssh-keygen and ssh-copy-id to copy the public key to the remote node.
